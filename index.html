Use Cases

What you can build

Real-world patterns from Domo + Snowflake customers

01

Unified Customer 360

Ingest CRM, marketing, and support data into Snowflake via Domo connectors. Join and enrich with Magic ETL. Build real-time customer dashboards.

-- Magic ETL pushes this to Snowflake
SELECT c.*, m.campaign_source, s.ticket_count
FROM customers c
JOIN marketing_touches m ON c.id = m.customer_id
JOIN support_summary s ON c.id = s.customer_id
02

ML Feature Engineering

Use Magic ETL to prepare training datasets on Snowflake. Write enriched features back to tables for ML pipelines. Version with Snowflake Time Travel.

-- Features written back to Snowflake
CREATE TABLE ml_features AS
SELECT customer_id,
  AVG(order_total) as avg_order,
  DATEDIFF('day', MIN(order_date), MAX(order_date)) as tenure
FROM orders GROUP BY 1
03

Real-time Operational Dashboards

Federated queries hit live Snowflake data. No caching delays. Monitor KPIs with up-to-the-second accuracy across your organization.

-- Federated query - always fresh
SELECT region, SUM(revenue) as daily_revenue
FROM SNOWFLAKE.LIVE.SALES
WHERE sale_date = CURRENT_DATE()
GROUP BY region
04

AI-Enriched Data Pipelines

Leverage Snowflake Cortex functions in your Domo workflows. Summarize documents, classify text, extract entities—all within your existing pipelines.

-- Cortex AI in your pipeline
SELECT ticket_id,
  SNOWFLAKE.CORTEX.SUMMARIZE(description) as summary,
  SNOWFLAKE.CORTEX.CLASSIFY(description, ['bug','feature','question']) as category
FROM support_tickets
